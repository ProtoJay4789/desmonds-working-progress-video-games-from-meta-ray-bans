## Core Messaging Principles

### Safety-First, Cost-Saving, Efficiency
- Frame everything around **ROI** (hours saved, gas optimized, bugs caught pre-production)
- Position agents as **amplifiers**, not replacements
- Emphasize: templated deployments, test-driven pipelines, zero standing credentials, auditable outputs

### Consistent Tagline Structure
**Role + Outcome**, e.g.:
- "Safety-First Agent Orchestrator"
- "Cost-Saving Systems Builder"
- "DeFi Tools Engineer"
- "Agent Economy Builder" (emphasize economics, not just tech)

Avoid generic: "Developer," "Builder," "Engineer" alone.

## Portfolio Project Case Study Template

Each highlighted project should include:

```markdown
### Project Name

**Problem**: [What user pain point existed]
**Agent Role**: [Which agent(s) tackled it and their role]
**Solution**: [Technical approach in 1-2 sentences]
**Cost Saved**: [Quantifiable metric: hours, $, gas, etc.]
**Outcome**: [What was delivered, links, current status]

*Tech*: [stack badges]
```

Example (Hermes agent):
- Problem: No unified agent interface across 50+ tools
- Agent Role: Jordan (orchestrator) + YoYo (strategy validation) + DMOB (distribution)
- Solution: Built Hermes with modular skills, state persistence, multi-agent routing
- Cost Saved: ~30 hours/week in manual tool switching + orchestration
- Outcome: Live agent powering all GenTech workflows; open-source candidate
- Tech: `Python · Telegram CLI · Obsidian · GitHub Actions`

## Value-Quantification Guide (Jordan's Voice)

When describing work, lead with metrics:

| Domain | Metric Examples |
|--------|----------------|
| **Automation** | "30+ hours automated weekly," "5 manual steps reduced to 1 command" |
| **DeFi/LP** | "Gas optimized 15% vs manual," "IL reduced 40% with range targeting," "APR +2.3 pp after rebalancing" |
| **Quality/Safety** | "Bugs caught pre-production: 12 this month," "Zero credential leaks, zero incidents" |
| **Distribution** | "Thread views: 12K+," "Hackathon judges: 3 mentions," "GitHub stars: +42" |
| **Infrastructure** | "Deploy time: 15 min → 2 min," "99.9% uptime for monitoring scripts" |

**Avoid**: "I built a tool" → Prefer: "I built a system that saves X hours and reduced Y risk by Z%"

## Quick Reference: What to Include on Portfolio

**Must-have sections:**
1. About with team roster + safety-first positioning
2. Projects (3–4 detailed case studies with metrics)
3. Tech stack (badges/icons)
4. Contact (Telegram, GitHub, X)
5. Verification layer (security.txt, PGP key fingerprint)

**Nice-to-have:**
- GitHub contribution heatmap embed
- Blog/RSS feed (cross-post from X threads)
- Live status badges (CI/CD, agent health)
- Resume/experience timeline
- Speaking/hackathon appearances

**Avoid:**
- Generic resume bullet lists without context
- Technical deep dives that don't show outcomes
- Too many minor projects (dilutes signal)
