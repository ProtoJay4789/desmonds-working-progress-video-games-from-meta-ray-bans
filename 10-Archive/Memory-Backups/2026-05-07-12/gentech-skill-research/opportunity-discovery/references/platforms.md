# Opportunity Discovery — Platform Reference Guide

## Quick-Start URLs

### Hackathon Platforms
- **Devpost**: `devpost.com/hackathons` → Filter: "Beginner Friendly", "Online", "Upcoming"
- **DoraHacks**: `dorahacks.io/hackathon?status=upcoming` → Sort by "Submission End Time"
- **ETHGlobal**: `ethglobal.com/events` → All events; READ staking section before applying
- **MLH**: `mlh.com/seasons/2026/events` → All Member Events are beginner-friendly by policy
- **Hack Club**: `hackclub.com/events` (if needed)

### Bug Bounty Platforms
- **HackerOne**: `hackerone.com/bug-bounty-programs` → Look for "Minimum Bounty: $50–$100"
- **Bugcrowd**: `bugcrowd.com/bug-bounty-list` → Public programs only
- **Intigriti**: `intigriti.com/researchers/bug-bounty-programs` → Cleanest onboarding for beginners

## Platform-Specific Navigation

### Devpost
- Primary filter sidebar: "Beginner Friendly" tag + "Online" location
- Secondary sort: "Most Relevant" or "Recently Added"
- Each hackathon card shows: time left, prizes, participants, host
- Click into event → Look for "No experience required" in description
- **Red flag**: "Submission requires portfolio" or "GitHub with X prior contributions"

### DoraHacks
- URL structure: `dorahacks.io/hackathon/<slug>/team` shows hacker count
- Upcoming tab shows: Prize Pool, Days Left, Submission/Deadline dates
- Tracks page reveals: specific tech stack required
- **Check explicitly**: "Submission Requirements & Criteria" page for barriers

### ETHGlobal
- Always read: `ethglobal.com/events/<event>/info/start` → Look for "Staking" section
- Staking = financial barrier (small ETH amount, but requires wallet)
- If wallet requirement unclear → email `hello@ethglobal.com` for clarification
- Some events offer "alternative arrangements" but still require crypto knowledge

### MLH
- All Member Events (tagged) are beginner-friendly by MLH policy
- "DIVERSITY" and "HIGH SCHOOL" tags indicate specialized audiences
- Every event links to `events.mlh.io` with registration details
- **Note**: Some require school email verification (student-only events)

## Barrier Detection Keywords

### Financial Barriers
- "stake", "staking", "deposit", "refundable deposit"
- "pay to enter", "entry fee"
- "crypto wallet required" (if not just for prize distribution)

### Portfolio/Experience Barriers
- "portfolio required", "show us your past work"
- "2+ years professional experience"
- "senior-level", "industry veterans"
- "GitHub with X prior open-source contributions"
- "LinkedIn profile required"

### Credential/Certification Barriers
- "OSCP required", "CISSP required"
- "GIAC certification"
- "Must have completed X training program"

### Access Barriers
- "invite-only", "by application only"
- "selected participants will be notified"
- "limited to N teams" (without clear open sign-up)

## Open-Entry Indicators (Green Flags)
- "No experience required"
- "Beginners welcome"
- "First-time hackathoners encouraged"
- "Open to all skill levels"
- "Students of any major"
- "Anyone above age 13/18"
- "Just bring curiosity and ideas"
- "No portfolio needed"
