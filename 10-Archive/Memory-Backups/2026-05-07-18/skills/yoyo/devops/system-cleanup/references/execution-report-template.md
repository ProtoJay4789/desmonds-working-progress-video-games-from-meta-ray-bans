## Template Structure

### File 1: Cleanup Plan (pre-execution)
`10-Archive/cleanup-manifests/YYYY-MM-DD-Spring-Cleaning-Plan.md`

Contains:
- Audit findings table (offender, size, action)
- Risk categorization (A/B/C)
- Execution plan with command snippets
- Before/After projections
- Items explicitly NOT to touch

**Purpose:** Team review → approval → sign-off before destructive operations

---

### File 2: Execution Report (post-execution)
`10-Archive/cleanup-manifests/YYYY-MM-DD-Execution-Report.md`

Contains:
- ✅ Phase-by-phase status (COMPLETE/DEFERRED/FAILED)
- Verification outputs (raw command output snippets)
- Before/After comparisons (table)
- Cumulative results (GB freed, % disk recovered)
- Docker phase notes (if deferred)
- Telegram summary sent (for coordination record)

**Purpose:** Audit trail + runbook for next cleanup cycle

## Lessons from 2026-05-02 Session

1. **Always check nested dot-dir depth first** — recursive backup trap detection is Step 0 before any size-based deletion
2. **Preserve HuggingFace models at all costs** — 142M is small but re-download is hours; add to `NEVER_DELETE` list in skill
3. **NPM cache clean fails under nvm** — either `source ~/.nvm/nvm.sh` first or skip; document in skill pitfalls
4. **Journalctl vacuum may report 0B freed** — if already minimal, skip to save time
5. **Docker 0B reclaimable = all images active** → requires service stop; don't waste time pruning live containers
6. **Manifest vaulting BEFORE deletion** — provides rollback reference and satisfies audit requirement
