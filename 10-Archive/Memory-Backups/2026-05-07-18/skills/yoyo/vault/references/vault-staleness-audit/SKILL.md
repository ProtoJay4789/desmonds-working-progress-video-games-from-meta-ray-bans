# Vault Staleness Audit

**Systematic health check for the Gentech vault — detect aging documents, broken workflows, and content that needs archiving or refresh.**

---

## When to Run

- Scheduled cron job (weekly or before sprint planning)
- After major hackathon phases to prune resolved items
- When onboarding new agents — establish clean baseline
- If daily sync backlog exceeds 2 days

---

## Trigger Conditions

Any of these warrants an audit:
- `00-Working-Memory.md` last modified > 5 days ago
- Missing `08-Daily/` notes for > 2 consecutive calendar days
- Active handoffs in `09-Green Room/active-handoffs/` aging > 3 days
- Pending items on `11-Mess Hall/handoff-board.md` aging > 7 days
- Master todo (`09-Green Room/master-todo.md`) date header doesn't match current sprint

---

## Step-by-Step Procedure

### Phase A — Core Metadata Scan
1. Check Working Memory `00-Working-Memory.md`: file mtime + `Last Updated:` header
2. Scan `08-Daily/`: list files, detect gaps vs expected date range (last 7 days)
3. Inspect `09-Green Room/active-handoffs/`: list files, compute age (days since creation)
4. Read `09-Green Room/master-todo.md`: verify date header matches current sprint

### Phase B — Content Drift Analysis
5. Cross-reference aging handoffs with `11-Mess Hall/handoff-board.md` and `agent-coordination-board.md`
6. Search Mess Hall archive (`11-Mess Hall/archive/`) for completed items that should be purged
7. Scan `00-Inbox/` for files > 7 days old (stale intake)
8. Review `00-HQ/Approvals/` for unactioned approvals (> 3 days)

### Phase C — Specific Failure Modes
9. **Missing daily notes:** Backfill from `11-Mess Hall/daily/<week>/` daily context entries
10. **Stale handoffs:** Classify as BLOCKED (dependency), ABANDONED, or DEADLINE-RESET needed
11. **Placeholder scan:** Grep for `TODO`, `FIXME`, `0x000`, `placeholder`, `PLACEHOLDER` in code/config notes
12. **Duplicate/scatter:** Identify same-topic files fragmented across Green Room that need consolidation

### Phase D — Flagging Matrix
13. For each finding, compute severity score:
    - 🔴 **P0** — blocks production/ship (DVN placeholder, unapproved SC variants)
    - 🟡 **P1** — needs human decision within 48h (stale handoffs > 7d, WM outdated > 5d)
    - 🟢 **P2** — cleanup/archival (resolved items, temp files, duplicate docs)
14. Limit final report to **top 5 items** by (severity × age) product
15. For each item: path, age, issue summary, evidence snippet, recommended action

```
# Vault Health Report — [DATE]

**Health Score:** X/10 | Items scanned: N | Stale: 5

   - **Evidence:** Specific lines, date mismatches, placeholder markers
   - **Action:** What needs to happen (ARCHIVE / REFRESH / HANDOFF-DROP / PLACEHOLDER-REPLACE)

...
```

## Pitfalls

- **False positive on archived items:** Always check `10-Archive/` before flagging; some old files are intentional references
- **Time zone confusion:** All ages computed against vault server time (UTC); convert to agent local time for deadline context
- **Cron vs manual:** Some handoffs auto-generated by cron jobs (LP tracker, D5 milestone); distinguish from human handoffs
- **Active vs passive placeholders:** `Kite DVN placeholder` = real gap; `Gas Reserve monitoring trigger` = pending strategy task — treat differently
- **Over-flagging:** Only flag items requiring human intervention; auto-fixable items belong in separate cleanup pass
- **Daily note location:** Day notes live in `08-Daily/` not `11-Mess Hall/daily/` — don't confuse archives with active sync failures
