## Overview

When applying for crypto-native companies (DeFi protocols, trading venues, infrastructure projects), traditional resumes are secondary to **public, production-grade work**. This skill governs the full application workflow: portfolio evaluation → narrative framing → material preparation → interview strategy for roles where GitHub commits and on-chain deployments speak louder than degrees.

---

## When to Trigger

Apply this skill when:
- User is pursuing **internship, full-time, or contract roles** at crypto companies
- The target is **remote-first, DeFi-native, or trading-focused** (e.g., Hibachi, No Limit Holdings, Uniswap Labs)
- The role emphasizes **technical growth, protocol engineering, or go-to-market engineering**
- User has **production code on-chain or open-source projects** but limited formal resume experience

**Do NOT trigger** for traditional finance (TradFi) or non-tech roles where resume/LinkedIn is still primary.

---

## Core Workflow

### 1. Portfolio-First Assessment
Before any application materials are drafted, audit the user's portfolio with these questions:
- **Production presence:** Are any projects already live on mainnet? (e.g., AgentEscrow, AAE Dashboard)
- **Hackathon momentum:** Are they actively building in public competitions? (ETHGlobal, Solana Frontier, Kite AI)
- **Tech stack match:** Do they have Solidity + Python combinations? (critical for growth engineering)
- **Ecosystem signals:** GitHub contributions graph, fork network, open-source collaboration

**Signal thresholds:**
- ≥1 live mainnet deployment = STRENGTH
- ≥2 active hackathons in progress = MOMENTUM
- Solidity + Python combo = GOLD STANDARD for DeFi roles

### 2. Narrative Framing (Position as "Already Doing the Job")

For crypto-native hiring, frame the candidate as:
> "I'm already building the infrastructure this role would use — I'm just looking to scale it with your team."

**Key narrative pillars:**
- **Production credibility:** "My AgentEscrow is live on Avalanche with real users"
- **Hackathon velocity:** "I'm concurrently building three AAE projects while competing in ETHGlobal"
- **Stack alignment:** "I write smart contracts in Solidity and automate data pipelines in Python — exactly what your SQL/Python growth work needs"
- **Ecosystem embeddedness:** "I'm deep in the Avalanche/Solana ecosystems, not just studying them"

**For role-specific asks (like "Claude expert for SQL/Python"):**
- Tie the portfolio project directly: "My AAE Milestone Tracker automates on-chain data analysis using Python and API calls similar to Claude-assisted workflows"
- Show, don't tell: link to the repo/code rather than listing skills

### 3. Application Material Strategy

**Resume alternative:** Use the GitHub portfolio as the primary credential. If a resume is requested:
- Create a **one-page "Crypto Resume"** that is essentially a portfolio highlight reel
- Front-load production projects with live links and on-chain txn hashes
- Include hackathon track as "Recent Work" section
- Technical skills table: Solidity, Python, DeFi protocols, MCP integrations

**Cover letter approach:**
- Opening: "After reviewing your growth role, I noticed you're looking for someone with SQL/Python automation experience — my AAE Milestone Tracker does exactly that for real on-chain positions"
- Middle: "I'm not just studying DeFi, I'm shipping in it — AgentEscrow is already handling X txns, and I'm actively competing in Y hackathons"
- Close: "My Amazon flex schedule gives me the flexibility to deliver growth work on US/EU timezones while maintaining my own DeFi projects"

### 4. Interview Prep Focus

For technical interviews at crypto companies:
- **Be ready to walk through production code:** Have 2-3 repos selected for deep dives
- **Know your on-chain metrics:** TVL, transaction counts, user addresses for live projects
- **Hackathon demos:** Have live demos or video walkthroughs ready for current projects
- **Ecosystem fluency:** Understand the target company's protocols (e.g., if Hibachi trades on Avalanche, know your AVAX DeFi landscape)
- **Python/SQL pairing:** Prepare to discuss data pipelines that combine on-chain data with off-chain analytics

---

## Pitfalls to Avoid

1. **Academic overemphasis:** Don't lead with coursework, degrees, or school projects. Crypto hiring cares about shipped code.
2. **Vague claims:** Never say "experienced in DeFi" without pointing to a live protocol or audit.
3. **Resume-only submissions:** If they ask for a resume, always attach the portfolio link prominently. The portfolio is the main story.
4. **Ignoring hackathon momentum:** Active hackathon participation is a SIGNAL of passion, not a distraction — frame it as concurrent shipping.
5. **Underplaying Amazon flex:** The remote-flex arrangement is a STRENGTH for internships — it means no relocation cost, immediate availability, and low turnover risk.
