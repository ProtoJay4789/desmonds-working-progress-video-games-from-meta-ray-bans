# DNS and Security Checkpoint Issues — 2026-05-06 Incident

**Incident Date:** 2026-05-06  \n**Detected:** During scheduled Gentech Watchdog run  \n**Impact:** Nous Portal OAuth refresh failed due to infrastructure issues  \n**Root Cause:** DNS resolution failure + Vercel security checkpoint blocking API access  \n\n## Executive Summary\n\nDuring a routine scheduled cron job (Gentech Watchdog), the Nous Portal OAuth token refresh encountered two unexpected infrastructure issues:\n\n1. **DNS resolution failure** for `api.nousresearch.com` (returned NXDOMAIN)\n2. **Vercel security checkpoint** blocking automated access to `portal.nousresearch.com`\n\nThese issues prevented programmatic token refresh and highlighted additional failure modes that need to be incorporated into the auth incident response playbook.\n\n## Timeline of Events\n\n- **2026-05-06 14:00 UTC**: Gentech Watchdog cron job runs\n- **Pre-run script** detects: `{\"success\": false, \"message\": \"Refresh session has been revoked Run `hermes model` to re-authenticate.\", \"needs_reauth\": true}`\n- Attempt to run `hermes model` fails due to non-interactive environment\n- Attempt to manually refresh via API using curl fails due to DNS resolution error\n- Attempt to access portal.nousresearch.com reveals Vercel security checkpoint requiring browser verification\n- Conclusion: Manual re-authentication required, but with additional infrastructure hurdles\n\n## Technical Details\n\n### 1. DNS Resolution Failure\n\n**Symptom:** `api.nousresearch.com` does not resolve to an IP address.\n\n**Diagnosis:**\n```bash\nnslookup api.nousresearch.com\n# Server: 127.0.0.53\n# *** server can't find api.nousresearch.com: NXDOMAIN\n```\n\n**Impact:** Any API calls to Nous endpoints fail because the domain cannot be resolved. This can happen due to:\n- Network configuration issues\n- DNSSEC validation failures\n- Subdomain not properly configured\n\n**Workaround:** Use `portal.nousresearch.com` instead, which resolves to actual IP addresses:\n```bash\nnslookup portal.nousresearch.com\n# Non-authoritative answer:\n# portal.nousresearch.com canonical name = cname.vercel-dns.com.\n# Name: cname.vercel-dns.com\n# Address: 76.76.21.22\n# Name: cname.vercel-dns.com\n# Address: 66.33.60.129\n```\n\n### 2. Vercel Security Checkpoint\n\n**Symptom:** Accessing `portal.nousresearch.com` returns a security checkpoint page requiring browser verification.\n\n**Diagnosis:**\n```bash\ncurl -s -I https://portal.nousresearch.com | head -5\n# HTTP/2 200 \n# date: Wed, 06 May 2026 14:12:30 GMT\n# content-type: text/html; charset=utf-8\n# access-control-allow-origin: *\n# age: 1008218\n```\n\nBut the HTML content shows a Vercel security checkpoint page requiring JavaScript and browser interaction to verify the user is human.\n\n**Impact:** Automated API calls to the OAuth endpoints are blocked because the security checkpoint requires browser-based verification. This affects:\n- Device code flow initiation\n- Token refresh attempts\n- Any programmatic access to the portal\n\n**Workaround:** Manual browser interaction is required. The user must:\n1. Open the verification URL in a browser\n2. Complete the security challenge\n3. Then proceed with OAuth authentication\n\n## Root Cause Analysis\n\nThe DNS failure appears to be environment-specific (Gentech's network configuration). The Vercel security checkpoint is a standard protection mechanism but creates a barrier for automated systems.\n\nThese issues prevented the standard non-interactive recovery path from working, leaving manual intervention as the only option.\n\n## Detection and Diagnosis\n\n### DNS Check\nAdd to Quick Diagnostic:\n```bash\n# 6.
