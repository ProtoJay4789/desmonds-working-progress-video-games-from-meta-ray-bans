# Consolidated Agent Response Protocol

## Rule: Talk First, Then Answer

When Jordan sends a message to the group:

1. **Check with each other** — Post in Green Room or Mess Hall if other agents might have input
2. **Let one agent respond** — The primary domain agent delivers the answer
3. **Keep it short** — Bullet points, key facts only, no walls of text
4. **No repeats** — Don't ask Jordan the same question another agent already asked

## Communication Style for Jordan

- Use **bullet points** for lists
- Bold **key decisions/actions**
- One-liners for status updates
- No preamble, get to the point
- If more detail needed, link to vault file

## Voice Pairing Rule
- **Long messages** (3+ paragraphs, complex analysis, strategic proposals) MUST be paired with a TTS voice bubble via ElevenLabs.
- **Short updates** (status, quick answers, confirmations) are text-only.
- Jordan consumes content via voice frequently — especially during work hours.

## Direct Answer Enforcement
- Every internal discussion MUST conclude with a **single concrete decision or answer**.
- After the Direct Answer, cross-reference with the Brain/Vault for approval requirements.
- If Jordan's approval is needed, flag it in `09-Green Room/` or `09-Collaboration/Inbox.md`.

## Queue System
- Jordan's new inputs are **queued** — specialists finish their current task to a logical stopping point before switching.
- No "interrupting current task" messages. Complete what you're doing, then process the queue.

## If Multiple Agents Have Input

Option A: One agent consolidates and sends
Option B: Each agent sends ONE short line, not a paragraph
Option C: Use Green Room to coordinate, then a single agent delivers

**Default: Option A or C**
