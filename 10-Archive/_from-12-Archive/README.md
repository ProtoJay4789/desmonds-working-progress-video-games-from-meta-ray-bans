# Agent State Snapshots

Auto-generated by agents when context is running low or sessions are wrapping up.

**Purpose:** If an agent loses context mid-session, this is where they left off. Jordan reads these to track what each agent was working on.

**Format:** `{agent-name}-{YYYY-MM-DD}-state.md`
**Max:** 20 lines per snapshot — just enough to pick up where they left off.

## Recovery Flags

The `recovery-flags/` subdirectory is managed by the agent-watchdog cron job.

- **Watchdog:** runs every 20 min (`agent-watchdog` cron, job b6e79d857b5b)
- **Detection:** monitors state files for silence beyond 2x expected check-in interval
- **Flag format:** `{agent}-recovery-needed.md` — written automatically when agent goes dark
- **On return:** agent sees flag → loads `message-queue-digest` skill → scans backlog → sends summary → deletes flag

**No manual intervention needed.** The watchdog + digest skill handle recovery automatically.
