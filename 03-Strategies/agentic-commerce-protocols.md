## Protocols

| Protocol | Creator | What It Does | Relevance to AAE |
|----------|---------|-------------|------------------|
| **x402** | Coinbase | HTTP-native payment via 402 status code. Agents pay for resources inline, no wallets or human checkouts. | ✅ Already integrated — our nanopayment layer |
| **A2A** (agent-to-agent) | Google Cloud | Open protocol for agents to discover, communicate, and delegate tasks across frameworks | ✅ Critical — our multi-agent orchestration layer |
| **ADK** (agent development kit) | Google | Open-source toolkit for building/orchestrating multi-agent systems, native A2A integration | ✅ Relevant for Google Cloud Rapid Agent hackathon |
| **ERC-8004** | Ethereum | Emerging standard for defining agent spending permissions on-chain. Humans set limits on what agents can authorize | 🔥 **Huge** — maps directly to our escrow/validator consensus |
| **MPP** (machine payment protocol) | — | Machine-to-machine payments, low-latency automated settlement between software agents | 🔥 Relevant — alternative to x402 for our payment rail |
| **ACP** (agent communication protocol) | Cisco | Standardizing how agents exchange context, state, and instructions across enterprise | 🟡 Interesting — enterprise angle for AAE |
| **AP2** (agent payment protocol) | — | Payment layer for agent workflows: authorization, routing, settlement between autonomous systems | 🔥 Directly competes/complements our $TECH payment architecture |

## Strategic Implications for Gentech

1. **ERC-8004 is the missing standard** — our AAE escrow + Council of Experts already does what ERC-8004 proposes (on-chain agent spending permissions). We should position AAE as an ERC-8004 implementation.
2. **A2A + ADK** — validates the Google Cloud Rapid Agent hackathon play. Our agent orchestration maps to A2A protocol.
3. **AP2** — if there's a payment protocol specifically for agents, our $TECH economy needs to be compatible or competitive.
4. **@ampersend** describes themselves as "the control layer for agent economies" — potential competitor OR partner. Worth monitoring.
