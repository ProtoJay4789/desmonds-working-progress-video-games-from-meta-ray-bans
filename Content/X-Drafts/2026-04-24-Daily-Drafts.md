## POST 1 — LayerZero Security (Hot Take / Thread)
**Tone:** Alarming but educational
**Best window:** 12-2 PM ET or 5-6 PM ET

**1/** 🧵 Your favorite cross-chain bridge might be running a 1-of-1 multisig and calling it "decentralized."

KelpDAO just got exploited through LayerZero's DVN configuration.

The problem? Most projects never changed the default.

**2/** Dune data on 2,665 OApps using LayerZero:
- ~47% run 1-of-1 DVN (one validator = full control)
- ~45% run 2-of-2 (two validators, both compromised = game over)
- Only ~5% run 3-of-3+ (actual security)

**3/** That means roughly 92% of LayerZero integrations are one key leak away from total bridge drainage.

Not a bug. A configuration choice.

**4/** Compare:
- LayerZero: user-configurable DVN (most pick 1)
- Wormhole: 19 guardians, 13/19 threshold
- Axelar: validator set

The "flexibility" narrative becomes the vulnerability when defaults are unsafe.

**5/** If you're holding tokens in a protocol that bridges cross-chain, check their DVN config.

If it's 1-of-1, you're not decentralized. You're trusting one server.

**6/** For builders: 3-of-3+ minimum. For users: verify before you bridge.

Interoperability is the future. But only if we stop pretending 1 validator = security.
