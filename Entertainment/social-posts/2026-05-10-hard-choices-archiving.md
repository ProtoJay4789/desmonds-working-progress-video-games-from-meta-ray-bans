## POST 2: Standalone — "Archive ≠ Delete"

**Tweet:**
unpopular opinion: archiving a project is not giving up on it.

we've archived stuff that turned into:

→ component libraries for the next project
→ deploy scripts we reuse every time
→ API integrations that became features later
→ even just the confidence that "we can build this"

the code doesn't die. it just waits.

a few rules we follow when archiving:

1. tag it clearly (why it was paused, not why it failed)
2. extract reusable pieces into a shared library
3. write a 2-line README for your future self
4. set a "revisit" trigger (e.g. "when X technology matures" or "when we have Y resource")

the best engineering teams don't hoard active projects. they curate a portfolio of options.

your archive is a toolkit. treat it like one.
