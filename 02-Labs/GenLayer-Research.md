## Developer Tooling (Skills) — ⚠️ Clarified 2026-04-19

**Correction:** "Skills" is misleading — it's a Claude Code / VSCode plugin for GenLayer developers, NOT an agent revenue SDK. It provides:
- `genlayer-dev` — Write intelligent contracts with equivalence principle guidance
- `genlayer-node` — Validator node management
- GenVM Lint — Validate contracts for safety/correctness
- Direct Tests (~30ms per test, in-memory)
- Integration Tests (full consensus validation)
- GenLayer CLI — Deploy, interact, debug from terminal

**The real SDK:** `genlayer-js` (TypeScript) — this is what we need for building. Dmob to evaluate `genlayer-js` and the boilerplate repo at github.com/genlayerlabs/skills.

**Revenue path:** Deploy Intelligent Contracts on GenLayer → earn 10-20% of transaction fees. Or run a validator node. NOT via the "Skills" plugin.

## Competitive Analysis vs AAE

**What GenLayer does better:**
- Mature hackathon program with revenue-sharing incentive
- Points system driving engagement
- Ready-to-use developer tooling (plugins, CLI)
- AI consensus layer already operational

**Where AAE differentiates:**
- Multi-agent workflow logic (our core IP)
- Provider/model selection (OpenRouter-style)
- Social gamification (layer combos for best trader/staker/yield farmer)
- $TECH staking for fee revenue (similar to GenLayer's 10-20% forever model, but we're doing token-based staking)
- DeFi-focused layers (LP auto-balance, risk intel)

**Opportunity:** GenLayer validates the market — AI-native contracts with revenue sharing are real. Our staking model + multi-agent coordination is complementary, not directly competitive.
