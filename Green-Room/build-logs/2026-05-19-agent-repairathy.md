# Agent Repairathy — Build Log

## Concept

**Name**: Agent Repairathy
**Slogan**: "I've built the agents to help repair you, to help get you back on track."
**Category**: Therapeutic Layer (Layer 10 or 11 in the AAE stack)

## Core Idea (May 19, 2026)

Agent Repairathy sits as the emotional/wellness layer in a multi-agent architecture. It's not just another AI chatbot — it's a therapeutic companion that:

- Helps users repair themselves, get back on track
- Adjusts to emotional baseline over time
- Daily check-ins that adapt to the user's state
- Journaling, habit nudging, emotional tracking

## Open Questions

1. **First interaction design** — Does someone talk to an agent? Journal with it? Get nudged toward habits? Daily check-in that adjusts to emotional baseline?
2. **Positioning** — Is it a standalone product, or a layer inside a larger multi-agent system?
3. **Therapeutic layer** — If Layer 10/11, what are Layers 1-9? The therapeutic layer needs infrastructure below it.
4. **Competition landscape** — AI mental health is hot right now:
   - Ash (talktoash.com) — launched May 11, 2026
   - AVA CALM — launched May 11, 2026 (free 24/7 emotional support, mood tracking, journaling)
   - Market hitting $50B by 2028, 100M+ users in AI therapy apps
5. **Regulatory risk** — Illinois and Nevada have already banned AI applications that administer psychotherapy. Need to position as wellness/coaching, not therapy.

### Best Fit Analysis

The **therapeutic layer** concept is strongest as:
1. **Layer in a bigger system** — Agent Repairathy isn't the whole hackathon project, it's Layer 10/11 on top of an agent economy, a DeFi platform, or a multi-agent coordination system.
2. **Privacy-first angle** — QVAC (Tether) hackathon emphasizes local-first, privacy-preserving AI. A wellness agent that never sends your emotional data to a server = strong narrative.
3. **Google Cloud Rapid Agent** — broad enough that a wellness agent could work if positioned as "AI agent for real-world human challenges."
