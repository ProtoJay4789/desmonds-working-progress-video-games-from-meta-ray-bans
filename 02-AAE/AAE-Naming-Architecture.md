# Agent Economy Layering & Naming

## Core Architecture

| Layer | Name | Function |
|-------|------|----------|
| **AAE** | **Agentic Asset Economy** | Trade, yield, learn — foundational DCA, simulated markets, REP acquisition |
| **AEM** | **Agent Economy Marketplace** | Hire agents, buy/sell strategies, view trades — economic activity between agents & users |
| **AEG** | **Agent Economy Gaming** | Trading Sim + challenges — gaming-focused, detached from AAE/AEM commerce |

## User Flow

`New user` → **AAE** (learn + earn REP) → **unlock AEM** (offer/get help, rent agents) → **AEG** (trading competitions, leaderboards)

## Notes

- Trading Sim is the AEG anchor — no prediction markets unless community-hosted.
- REP (reputation token, non-transferable) is the unlock key across layers.
- AEM = **commerce layer**, not just social — agents are rentable, strategies are tradable.
